# VKTransit Bridge

**VKTransit Bridge** — это асинхронный мост между ВКонтакте и Telegram. Проект предназначен для переноса сообщений из VK в Telegram-группу или канал с поддержкой текста, фото, голосовых сообщений и файлов, а также с указанием ссылки на исходное сообщение в VK.

## Описание

Проект решает задачу централизованного переноса контента из VK в Telegram без ручного копирования. Основная идея — запускать VK-часть как user-client и Telegram-часть как bot-client, чтобы получать сообщения из VK и отправлять их в Telegram с сохранением источника.

Архитектура строится вокруг асинхронного Python-стека:

* Telegram-клиент — на **Pyrogram**
* VK-клиент — на **vkbottle**

## Текущий функционал

* Получение сообщений из VK через user-longpoll.
* Пересылка текста из VK в Telegram.
* Поддержка фото, голосовых сообщений и документов.
* Добавление ссылки на исходное сообщение VK в подпись или текст.
* Отправка в заданный Telegram-чат по `TG_TARGET_CHAT_ID`.
* Загрузка файлов во временную директорию.
* Асинхронная архитектура на `asyncio`.

## Roadmap

* Очередь сообщений для сглаживания всплесков нагрузки.
* Дедупликация сообщений по `peer_id` и `message_id`.
* Поддержка обратного направления: Telegram → VK.
* Расширенная обработка вложений VK: видео, аудио, стикеры, альбомы.
* Сохранение состояния и истории обработанных сообщений в SQLite/PostgreSQL.
* Гибкие фильтры по чатам, авторам и типам вложений.
* Улучшенная обработка ошибок, retry/backoff и graceful shutdown.
* Логи в JSON-формате для удобной интеграции с мониторингом.

## Запуск и настройка

### 1. Подготовь переменные окружения

Создай файл `.env` рядом с проектом:

```env
TG_API_ID=123456
TG_API_HASH=your_telegram_api_hash
TG_BOT_TOKEN=123456:ABCDEF_your_bot_token
TG_TARGET_CHAT_ID=-1001234567890

VK_TOKEN=your_vk_user_token
VK_TARGET_PEER_ID=
DOWNLOAD_DIR=/tmp/vk_tg_bridge
CAPTION_PREFIX=VK
```

### 2. Установи зависимости

```bash
pip install pyrogram tgcrypto aiohttp vkbottle
```

### 3. Запусти локально

```bash
python vk_tg_bridge_async.py
```

### 4. Запуск через Docker

Собери образ:

```bash
docker build -t vktransit-bridge .
```

Запусти контейнер:

```bash
docker run --rm --env-file .env vktransit-bridge
```

### 5. Получение VK token

Для VK user-longpoll нужен user access token с правами на сообщения. Обычно его получают через OAuth-поток VK и затем передают в `VK_TOKEN`.

### 6. Полезные замечания

* `TG_TARGET_CHAT_ID` должен указывать на группу или канал, куда бот уже добавлен и имеет право писать.
* Если `VK_TARGET_PEER_ID` пустой, мост будет обрабатывать входящие события без дополнительной фильтрации.
